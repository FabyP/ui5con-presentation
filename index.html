  • Again, the google lighthouse audits were used, using a user centered approach of measauring it
  • This means:
  • Lighthouse doesn't look at when are all my files loaded, but when can the user see first content, when is he able to interact with my application and thats exactly what those metrics with their weight do.
  • By the weight you can see which ones are the most important ones, so first-conentful paint is when my user sees the content, interactive means when can he interact with the application and the speed index is the average loading time till UI Elements are displayed. So those three are the most important ones.
  • Again, we have a score from 1 to 100, with 100 as the best possible score
  • Any guessings? Which performance scored SAPUI5? ‚Ä¶ And React with the UI5 Web Components?

Performance

Score

SAPUI5

UI5 Web Components

  • Those are the results
  • SAPUI5 got a score of 30, while React with the UI5 Web Components got a score of 95. The results depend on your testing environment, if you wanna recreate this, your results may differ
  • In SAPUI5, it takes 5 seconds till we're able to see first content and we have to wait further 4 seconds to be able to interact with the application. Makes in total 9 seconds till the user can start effectively working with our Fiori App. And in our other application with React, whick looks completely the same and provides the same functionality, youve seen it, the site is fully loaded after 2,6 seconds.
  • The reason for this, is, that in SAPUI5, we have a ton of unused CSS that is loaded, even for controls we havent even used. And there is the SAP ui core library, which somehow blocks rendering.
  • On the contrary, we have a react, a framework that comes with much less overhead and as a developer, i was able to specify, which resources I want to have loaded in my application
  • This makes react extremely lightweight
  • Moreover, react uses a virtual dom, so if the state of one of my UI components changes, those actions are executed on a virtual dom. React then compares my changed virtual dom with my real dom and only changes those elements, which differ from the real dom
  • Maybe to add something here: We don't want to put SAPUI5 Controls in a bad light. We have seen it today. The UI5 Team is working hard on the performance, we have seen the performance improvements to make the core smaller and just use the controls and libraries which I really need. The performance is getting better.

  • I detected some more differences, for example the profile picture and logo in my shellbar. In react, my testsoftware wasnt happy with them, complaining about missing alternative texts. Other striking differences

  • Architecture
  • Need of CSS
  • There are two more differences I wanna mention at this point
  • In SAPUI5, we have a strict distribution into layout, functionality and model (MVC), while in react, we have all three of them combined in one component. This makes them reusable, but also worse to maintain the bigger my application gets, especially cause react lacks clearly defined best practices
  • And the other striking difference is the need off CSS in react, actually you need a ton of it, cause there arent much predefined components or layouts you can use like in SAPUI5
